# Breadcrumbs

Breadcrumbs show a user's current location within the page hierarchy and provide one-click access to parent pages. In the Cashfree dashboard they always appear directly above the PageHeaderset on internal pages, letting merchants quickly navigate back from a transaction detail to the transactions list without using the browser back button.

---

## When to use

| Use Breadcrumbs when... | Don't use Breadcrumbs when... |
|---|---|
| The user is 2+ levels deep in the hierarchy | The page is a top-level destination — there's nowhere to go back to |
| The path reflects a real navigable hierarchy | The "parents" are not actual linked pages |
| Used with PageHeaderset on internal pages | Used on modal overlays — there is no page hierarchy inside a modal |

---

## Anatomy

1. **Parent links** — Clickable `<a>` elements. Each navigates to the parent page.
2. **Separator** — A visual divider (chevron or slash) between each crumb.
3. **Current page** — Plain text (not a link). Represents where the user is now.

## Do / Don't

**Do:**
- Keep crumb labels concise — match the page's `<h1>` heading
- Always include a link for parent pages — they should be navigable
- Use Breadcrumbs together with PageHeaderset on internal pages

**Don't:**
- Use more than 5 crumbs without truncating the middle ones
- Include the current page as a link — it should be plain text
- Use Breadcrumbs as a primary navigation — they are a secondary wayfinding aid

---

## Accessibility

- Parent links are rendered as `<a>` elements — keyboard accessible natively
- The `<nav aria-label="breadcrumb">` wrapper identifies the landmark
- The current page item uses `aria-current="page"`
