# Button

Buttons are the primary mechanism for triggering actions in Cashmere. They communicate what will happen when a user interacts with them, and their visual weight signals importance within the page hierarchy. Three variants cover every action type — from the single most important CTA on a screen down to inline text links.

---

## When to use

| Use Button when... | Don't use Button when... |
|---|---|
| Triggering an action (submit, save, delete, confirm) | Navigating to another page — use a link instead |
| The primary call to action needs clear visual weight | There are more than 3 buttons on a single row |
| A destructive action requires an explicit user intent step | The action is part of a form with its own submit affordance |
| An icon alone isn't enough to communicate the action | You need to show multiple equal-weight options — use a ButtonGroup |

---

## Anatomy

1. **Container** — The clickable target. Minimum 32px height; recommended 44px+ on touch surfaces.
2. **Leading icon** (optional) — Reinforces the label. Placed 8px before the text.
3. **Label** — Sentence case, action-oriented text (e.g. "Save changes", not "OK").
4. **Trailing icon** (optional) — Suggests outcome (ChevronDown for menus, ExternalLink for new tab).

## Do / Don't

**Do:**
- Use sentence case for all button labels ("Save changes", not "SAVE CHANGES")
- Pair negative buttons with a confirmation step
- Always provide `aria-label` on icon-only buttons
- Keep labels short: 1–3 words is ideal

**Don't:**
- Place more than one primary button in the same visual section
- Disable a button without telling the user why — prefer hiding or a tooltip
- Use buttons for navigation — use `<a>` links instead
- Mix button sizes on the same row unless intentional (e.g. a large CTA next to a small dismiss)

---

## Accessibility

- All buttons render as native `<button type="button">` elements — keyboard focusable by default
- Icon-only variants require an `aria-label` prop; always provide one
- Disabled buttons are `disabled` on the DOM — they are excluded from tab order automatically
- Negative (destructive) buttons meet 4.5:1 colour contrast on the default light background
- Focus ring is visible in both light and dark backgrounds
